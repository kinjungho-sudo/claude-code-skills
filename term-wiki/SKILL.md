# 용어 위키 (Term Wiki)

## 이 스킬의 목적

사용자는 **비개발자**다. 개발·비즈니스 전문 용어를 몰라서 의사소통에 어려움을 겪는다.
그래서 대화 중 어려운 용어가 나오면 **그 자리에서 쉽게 풀어 설명하고, 그 정의를 Notion 위키에 누적 기록**해 둔다.
목표는 두 가지: ① 지금 대화에서 사용자가 이해하게 돕는다 ② 같은 용어가 다시 나와도 찾아볼 수 있게 사전을 쌓는다.

## 설치 전 준비 (사용자가 직접 설정)

이 스킬은 Notion에 기록하므로 **두 가지 준비**가 필요하다:

1. **Notion MCP 연결** — Claude Code에 Notion MCP가 연결돼 있어야 한다
2. **Notion 페이지 2개 생성** — 아래 두 페이지를 본인 Notion에 만들고 URL을 메모해둔다:
   - 개발/기술 용어 위키 페이지
   - 비즈니스/서비스 용어 위키 페이지

3. **SKILL.md 수정** — 아래 섹션의 page_id를 본인 페이지 ID로 교체한다

> Notion 페이지 ID 찾는 법: 페이지 URL에서 마지막 32자리 영숫자가 ID
> 예) `https://notion.so/My-Page-abc123def456` → ID는 `abc123def456`

## 어느 페이지에 기록하는가

아래 page_id를 **본인 Notion 페이지 ID**로 교체하세요:

| 용어 종류 | 페이지 | page_id |
|----------|--------|---------|
| **개발/기술** 용어 | 개발 용어 Wiki | `여기에-본인-개발위키-페이지ID` |
| **비즈니스/서비스** 용어 | 비즈니스 용어 Wiki | `여기에-본인-비즈니스위키-페이지ID` |

분류가 애매하면 맥락상 사용자가 어느 관점으로 들었는지로 정한다. 그래도 모호하면 개발 페이지에 둔다.

## 언제 발동하는가 (적극적으로)

다음 중 하나라도 해당하면 발동한다. 사용자가 요청하지 않아도 **먼저** 한다.

1. 대화에 사용자가 모를 만한 **개발 용어**가 등장 — 내가 처음 쓰거나 설명 없이 던졌을 때
2. 내가 판단하기에 **어려운 전문 용어** — 비개발자가 바로 이해하기 힘든 단어
3. 사용자가 **"그게 뭐야 / 무슨 뜻이야 / 모르겠어 / 어렵다"** 라고 표현
4. 개발 용어가 아니어도, **비즈니스·서비스·도메인 용어**인데 사용자가 잘 모를 것 같을 때
5. 사용자가 **특정 용어를 콕 집어 "설명해줘"** 라고 요청 — 이때는 그 용어를 설명하고 **반드시 기록까지** 한다
   - 예: `"'어노테이션'이 뭐야? 설명해줘"` → 어노테이션을 쉽게 설명 + 중복 확인 후 알맞은 위키에 기록

판단 기준은 "senior가 아니라 **비개발자 사용자 입장**에서 낯선가?"이다. 애매하면 기록하는 쪽으로 기운다.

## 기록 형식 (반드시 이 세 줄)

각 용어는 H3 제목 + 세 줄로 적는다.

```markdown
### 용어 이름 (English)
- **한 줄 뜻:** 비개발자도 이해할 한 문장 정의
- **비유:** 일상 속 익숙한 상황에 빗댄 설명
- **우리 프로젝트에서:** 지금 작업 중인 프로젝트에서 어디에 어떻게 쓰이는지
```

- 비유는 식당·지도·금고·세이브포인트처럼 **누구나 아는 일상 장면**으로
- "우리 프로젝트에서"는 현재 대화 맥락의 프로젝트에 연결한다. 연결점이 없으면 "(이 프로젝트에서 아직 안 쓰임)"이라고 솔직히 적는다

## 절차

### 1단계 — 대화에서 먼저 쉽게 설명
용어가 나온 그 답변 안에서, 위 세 줄 형식과 같은 눈높이로 사용자에게 바로 풀어 설명한다. 기록은 그다음이다(대화 흐름을 끊지 않는다).

### 2단계 — 중복 확인 (필수)
기록 전 **반드시** 대상 페이지를 `notion-fetch`로 읽어 같은 용어가 이미 있는지 확인한다.
- 이미 있으면 → **기록하지 않는다.** "이미 위키에 있어요"라고만 짧게 알린다.
- 기존 설명이 비어 있거나 명백히 틀렸으면 `update_content`로 그 항목만 고친다(중복 추가 금지).

> 같은 용어 = 이름이 같거나 사실상 동의어(예: "env var" = "환경변수"). 한 페이지 안에서만 중복을 따진다.

### 3단계 — 알맞은 주제 아래에 추가
중복이 아니면 `notion-update-page` + `command: insert_content` + `position: {type: end}`로 추가한다.
- 페이지에 주제 섹션이 있으면 의미가 맞는 섹션 끝에 넣는 게 이상적이다. 섹션 경계 삽입이 까다로우면 페이지 맨 끝에 붙여도 된다.

### 4단계 — 사용자에게 한 줄 보고
"📒 '○○' 용어를 [개발/비즈니스] 위키에 기록했어요" 정도로 짧게 알린다.

## 도구

Notion MCP 도구는 deferred 상태일 수 있다. 없으면 `ToolSearch`로 먼저 로드한다:
`select:mcp__claude_ai_Notion__notion-fetch,mcp__claude_ai_Notion__notion-update-page`

## 하지 말 것

- 사용자가 이미 아는 게 분명한 쉬운 단어까지 기록하지 않는다(노이즈)
- 한 번에 사전 전체를 새로 쓰지 않는다 — 대화에서 **실제로 나온** 용어만 하나씩 쌓는다
- 중복 확인 없이 바로 추가하지 않는다
- 전문용어를 설명 없이 그냥 던지지 않는다. 비개발자가 듣는다는 걸 늘 기억한다
